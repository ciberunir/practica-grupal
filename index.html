Conceptos Básicos

¿Qué es el Phishing?

El phishing es una técnica de ingeniería social utilizada por atacantes para engañar a las víctimas y obtener información confidencial como credenciales de acceso, datos bancarios o información personal. Los atacantes se hacen pasar por entidades legítimas para ganar la confianza de sus víctimas.

¿Qué es WiFi Pineapple?

WiFi Pineapple es un dispositivo de auditoría de redes inalámbricas desarrollado por Hak5. Está diseñado para realizar pruebas de penetración en redes WiFi y puede ser utilizado para:

  • Crear puntos de acceso falsos (Evil Twin)
  • Interceptar tráfico de red
  • Realizar ataques Man-in-the-Middle (MitM)
  • Implementar portales cautivos maliciosos

¿Qué es un Portal Cautivo?

Un portal cautivo es una página web que se muestra a los usuarios cuando intentan acceder a Internet a través de una red WiFi pública. Normalmente se utiliza en hoteles, aeropuertos y cafeterías para:

  • Autenticar usuarios
  • Mostrar términos y condiciones
  • Recopilar información del usuario

En un ataque de phishing, este portal se utiliza para simular páginas de login legítimas y capturar credenciales.

Evil Twin Attack

El ataque Evil Twin consiste en crear un punto de acceso WiFi falso que imita a uno legítimo. Cuando las víctimas se conectan a este punto de acceso malicioso, todo su tráfico pasa por el dispositivo del atacante, permitiendo:

  • Capturar credenciales
  • Interceptar comunicaciones
  • Inyectar contenido malicioso
  • Redirigir a portales cautivos falsos

Prueba de Concepto

1

Configuración del WiFi Pineapple

El primer paso es configurar el dispositivo WiFi Pineapple. Esto incluye:

  • Conectar el dispositivo a una fuente de alimentación
  • Acceder a la interfaz de administración (normalmente en 172.16.42.1:1471)
  • Configurar las credenciales de acceso
  • Actualizar el firmware si es necesario
2

Creación del Punto de Acceso Falso

Configurar un punto de acceso que imite una red WiFi legítima:

  • Seleccionar un SSID común (ej: "WiFi_Gratis", "Starbucks_WiFi")
  • Configurar el canal de transmisión
  • Deshabilitar la autenticación para facilitar las conexiones
  • Activar el módulo de Evil Portal
3

Diseño del Portal Cautivo

Crear una página de login que simule un servicio legítimo:

  • Diseñar una interfaz idéntica al servicio objetivo (ej: Gmail, Facebook)
  • Incluir logos y estilos oficiales
  • Crear formularios de captura de credenciales
  • Configurar la redirección después del "login"
<form action="capture.php" method="POST">
    <input type="email" name="email" placeholder="Correo electrónico">
    <input type="password" name="password" placeholder="Contraseña">
    <button type="submit">Iniciar sesión</button>
</form>
4

Captura de Credenciales

Configurar el script de captura para almacenar las credenciales:

  • Crear un script PHP o Python para procesar los datos
  • Almacenar las credenciales en un archivo de log
  • Redirigir al usuario a la página real del servicio
<?php
// Ejemplo de captura (solo para fines educativos)
$log = fopen("credentials.txt", "a");
fwrite($log, "Email: " . $_POST['email'] . "\n");
fwrite($log, "Password: " . $_POST['password'] . "\n");
fwrite($log, "IP: " . $_SERVER['REMOTE_ADDR'] . "\n");
fwrite($log, "---\n");
fclose($log);
header("Location: https://www.google.com");
?>
5

Ejecución del Ataque

Proceso de ejecución del ataque:

  • Posicionar el dispositivo en un lugar con alta afluencia
  • Activar el punto de acceso y el portal cautivo
  • Esperar a que las víctimas se conecten
  • Monitorizar las conexiones y credenciales capturadas

⚠️ Advertencia Legal

Este contenido es exclusivamente educativo. Realizar ataques de phishing sin autorización es ilegal y puede conllevar graves consecuencias legales. Solo debe utilizarse en entornos controlados con autorización expresa.

Variaciones del Ataque

1. Ataque con Deautenticación

Esta variación combina el Evil Twin con ataques de deautenticación para forzar a los usuarios a desconectarse de la red legítima:

  • Se envían paquetes de deautenticación a la red objetivo
  • Los dispositivos se desconectan automáticamente
  • Al reconectarse, encuentran el punto de acceso falso con mejor señal
  • Mayor probabilidad de que se conecten al Evil Twin

2. Karma Attack

El ataque Karma aprovecha el comportamiento de los dispositivos WiFi:

  • Los dispositivos envían "probe requests" buscando redes conocidas
  • WiFi Pineapple responde a todas estas solicitudes
  • El dispositivo se conecta automáticamente creyendo que es una red conocida
  • Funciona especialmente bien con dispositivos que tienen redes guardadas

3. Portal Cautivo con Captcha

Añadir elementos de verificación para dar mayor credibilidad:

  • Incluir captchas de Google u otras verificaciones
  • Simular verificación en dos pasos
  • Añadir mensajes de "verificación de seguridad"
  • Mayor apariencia de legitimidad

4. Ataque Dirigido a Empresas

Personalizar el ataque para organizaciones específicas:

  • Investigar los sistemas de autenticación de la empresa
  • Clonar portales internos de la organización
  • Usar SSIDs que imiten las redes corporativas
  • Capturar credenciales de acceso empresariales

5. Combinación con SSL Strip

Degradar conexiones HTTPS a HTTP para interceptar datos:

  • Configurar un proxy transparente
  • Interceptar solicitudes HTTPS
  • Servir versiones HTTP de las páginas
  • Capturar credenciales en texto plano

6. Inyección de JavaScript

Inyectar código malicioso en páginas web legítimas:

  • Interceptar tráfico HTTP
  • Inyectar scripts de keylogger
  • Capturar todas las pulsaciones de teclado
  • Enviar datos a servidor controlado por el atacante

Cómo Protegerse

🔐

Usar VPN

Utilizar una VPN (Red Privada Virtual) cifra todo el tráfico de red, impidiendo que un atacante pueda leer la información interceptada.

🔒

Verificar HTTPS

Siempre verificar que las páginas de login utilicen HTTPS y que el certificado sea válido. Buscar el candado en la barra de direcciones.

📱

Desactivar WiFi Automático

Desactivar la conexión automática a redes WiFi. Conectarse manualmente solo a redes conocidas y de confianza.

🔑

Autenticación de Dos Factores

Activar 2FA en todas las cuentas. Incluso si capturan las credenciales, necesitarán el segundo factor para acceder.

⚠️

Desconfiar de Redes Públicas

Evitar introducir credenciales sensibles en redes WiFi públicas. Usar datos móviles para operaciones críticas.

🗑️

Eliminar Redes Guardadas

Eliminar regularmente las redes WiFi guardadas en el dispositivo para evitar conexiones automáticas no deseadas.

🔄

Mantener Software Actualizado

Mantener el sistema operativo y aplicaciones actualizadas para beneficiarse de las últimas correcciones de seguridad.

🛡️

Usar Gestores de Contraseñas

Los gestores de contraseñas no autocompletarán credenciales en sitios de phishing ya que verifican el dominio exacto.

Señales de Alerta

  • Redes WiFi sin contraseña en lugares públicos
  • Portales de login que aparecen inesperadamente
  • URLs sospechosas o sin HTTPS
  • Certificados de seguridad inválidos
  • Solicitudes de información excesiva
  • Errores ortográficos o de diseño en páginas de login
